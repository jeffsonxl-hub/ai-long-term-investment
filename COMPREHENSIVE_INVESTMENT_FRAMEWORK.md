# A 股长期投资综合分析框架（综合版）

> 一句话总结：做好 A 股长期投资，核心不是选股技巧，而是建立一套系统性的分析流程——让数据说话，让规则做决策，让情绪靠边站。

---

## 一、核心投资哲学

两条来源（Codex 系统分析 + ChatGPT 长期投资对话）高度一致，共同形成以下原则：

**原则一：买股票 = 买企业。** 不要像短线交易一样天天看 K 线，而应该像一个企业投资人，重点研究公司质量 + 行业前景 + 估值是否合理。

**原则二：安全边际是长期收益的来源。** 好公司买太贵也可能赔钱。在 A 股，PE 处于历史 30% 分位以下逐步建仓是较好的策略。

**原则三：复利来自时间，不来自交易。** 长期投资最大的敌人不是选错股，而是在大跌时拿不住。指标的意义不仅是选股，更是建立"我的公司基本面没问题"的信心。

**原则四：AI 的角色是缩小范围，不是替你做决策。** 约 5300 只 A 股 → 500 家优秀公司 → 100 家值得跟踪 → 20 家重点研究 → 最终由你决定。

### A 股特有的注意事项（Codex 补充）

A 股有几个区别于美股的"特殊国情"：
- 利润调节手段丰富：必须交叉验证净利润与现金流、扣非净利润
- 政策敏感性极强：政策就是最大的基本面
- 大股东行为影响巨大：质押比例、减持频率是重要信号
- 公司治理参差不齐：商誉暴雷、关联交易是常见风险

## 二、三层筛选漏斗

这是 Codex 提出的实操框架，可与 ChatGPT 的 AI Agent 系统无缝对接：

**第一层：粗筛（全市场扫描）**
5300+ A 股 → 500 家初选池

硬门槛（一票否决制）：
- ROE 连续 5 年 > 15%
- 经营现金流 / 净利润 > 0.7
- 商誉 / 净资产 < 20%
- 大股东质押 < 30%
- 无 ST / *ST / 退市风险
- 审计意见为"标准无保留"

对应 Agent：Company Agent + Financial Agent（快速打分）

**第二层：精筛（深度研究）**
500 家 → 100 家核心跟踪池

核心分析：
- 商业模式是否可持续（护城河宽度）
- 行业赛道是否"坡长雪厚"
- 管理层是否诚信且有能力
- 估值是否在历史合理区间

对应 Agent：Growth + Cash Flow + Valuation + Industry + Governance + Risk

**第三层：决策（择时 & 组合）**
100 家 → 20 家重点持仓 + 30 家观察名单

- 当前估值分位数是否在 30% 以下？
- 技术面是否出现企稳信号？（200 日均线、成交量）
- 北向资金/社保基金是否在加仓？
- 组合层面：行业集中度、单只仓位、最大回撤预期

对应 Agent：Technical + Macro + Portfolio Manager → 最终由投资者决策

## 三、指标体系全景（~110 个指标，12 大维度）

每个指标标注来源：C = Codex 首次提出，G = ChatGPT，C+G = 双方重合或互补。

### 3.1 盈利能力（Profitability）— 权重 20%

| # | 指标 | 推荐标准 | 来源 |
|---|------|----------|------|
| 1 | ROE | 连续 5 年 > 15%，优秀 > 20% | C+G |
| 2 | ROE 稳定性（近 5/10 年标准差） | < 5% | C+G |
| 3 | ROA | > 8% | G |
| 4 | ROIC | > 15%，且持续 > WACC | C+G |
| 5 | 毛利率 | 稳定或逐年提升 | C+G |
| 6 | 毛利率稳定性（近 5 年趋势） | 不下滑 | C |
| 7 | 净利率 | 稳定，> 行业均值 | C+G |
| 8 | 扣非净利润 / 归母净利润 | > 0.9（接近 1 为佳） | C |
| 9 | EBIT Margin | > 15% | G |
| 10 | EBITDA Margin | > 20% | G |

C 的独特贡献：扣非净利润 / 归母净利润比值。A 股很多公司净利润好看但扣非很差（靠卖资产/政府补贴），这个指标是去伪存真的关键。

| 17 | FCF 3 年 CAGR | > 0，持续提升 | G |
| 18 | 研发投入 / 营收 | 科技类 > 5%，不逐年下降 | C |
| 19 | 分析师一致预期营收增速 | 参考，不盲信 | G |
| 20 | 分析师一致预期 EPS 增速 | 参考，不盲信 | G |

C 的独特贡献：(1) 营收增速与利润增速匹配度——利润涨但营收不涨通常是降本红利而非真正成长；(2) 研发投入占比趋势——研发费率逐年下降可能是公司在吃老本。

### 4.2 评分等级

| 分数 | 评级 | 建议动作 |
|------|------|----------|
| 85-100 | 卓越 | 重点持仓，择机加仓 |
| 70-84 | 优秀 | 持仓跟踪，等待更好买点 |

| 40-54 | 一般 | 暂不参与，等待基本面改善 |

### 4.3 A 股特殊加减分规则

| 条件 | 加减分 | 说明 |
|------|--------|------|
| 扣非/归母 < 0.7 | -10 分 | 利润过度依赖非经常性损益 |
| 大股东质押 > 50% | -15 分 | 一票否决级别风险 |
| 商誉/净资产 > 30% | -10 分 | 减值暴雷高风险 |
| 被立案调查 | -20 分 | 立即回避 |
| 审计意见非标 | -20 分 | 财务可信度存疑 |
| 社保基金+北向持续加仓 | +5 分 | 聪明钱背书 |
| 行业被政策打压 | -15 分 | A 股特色系统性风险 |

## 五、AI 投资 Agent 架构设计

ChatGPT 提出的多 Agent 架构很完整，Codex 补充了 A 股特有的数据输入和漏斗流程。综合后的三层架构：

`
                         你（投资者）
                              |
                     Chief Investment Agent
                     （协调 + 最终报告生成）
                              |
         ┌────────────────────┼────────────────────┐
         |                    |                    |
    Research Layer       Portfolio Layer      Execution Layer
    （研究分析）           （组合管理）          （执行辅助）
         |                    |                    |
    ┌────┴────┐         Portfolio Manager    Timing Agent
    |         |         * 仓位建议            * 买卖时机
    | 质量:   |         * 集中度控制          * 分批建仓
    | Company |         * 组合风险评估        * 止损止盈
    | Growth  |         * 再平衡建议
    | CashFlow|              |
    |         |         Strategy Agent
    | 风险:   |         * 投资假设验证
    | Financial|        * 持有/卖出决策
    | Risk    |         * 情景分析
    | Governance|
    |         |
    | 环境:   |
    | Industry|
    | Macro   |
    | Valuation|
    | News    |
    | Technical|
    └─────────┘
         |
    数据层（Data Layer）
    AkShare /  / 东方财富 / 巨潮资讯
`

### Agent 设计示例：Cash Flow Agent

| 属性 | 内容 |
|------|------|
| Identity | 你是现金流侦探——利润可以被粉饰，但现金流不会说谎 |
| Goal | 验证公司利润的真实性和现金创造能力 |
| Input | 近 5 年现金流量表 |
| Output | Cash Flow Score + 利润含金量评估 + 核心发现 |
| Key Indicators | OCF/NI、FCF、收现比、OCF 增长、FCF Margin |
| A-Share Special | 收现比是最核心的 A 股利润验证指标 |

（其他 Agent 设计同理——完整定义见后续代码实现阶段。）

## 七、实施路线图

### Phase 1：数据基础
打通数据管道，能获取 A 股全市场基础数据。
- 搭建 Python 项目骨架
- 集成 AkShare，实现基础数据获取（股票列表、财报、行情）
- 数据缓存层（SQLite）+ 质量检验

### Phase 2：指标体系
实现 ~110 个指标的计算引擎。
- 财务指标计算器、现金流指标计算器、估值指标计算器
- A 股特色指标（扣非/归母比、商誉比例、大股东质押）
- 指标趋势分析（5 年趋势、变动方向）

### Phase 3：评分模型
实现 100 分综合评分模型。
- 各维度打分函数 + A 股加减分规则引擎
- 综合评分计算器 + 历史追踪

### Phase 4：Agent 框架
搭建多 Agent 分析框架。
- Agent 基类设计 + 核心 Agent 实现
- Chief Investment Agent（协调 + 报告生成）
- Portfolio Manager Agent（组合管理）

### Phase 5：前端展示
可视化展示分析结果。
- 个股综合评分卡（Morningstar 风格）
- 行业对比雷达图 + 全市场筛选器
- 持仓组合仪表盘

### Phase 6：自动化运营
